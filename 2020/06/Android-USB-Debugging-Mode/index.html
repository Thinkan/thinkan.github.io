Android开启调试模式

2020-06-15

在Android开发过程中，开启移动设备的调试模式可以帮助我们快速运行和调试程序。

USB调试

注意！如果是Android开发板，要选择OTG USB接口，注意参照一下板子的规格书。

常规设置

在电脑和移动设备使用USB连接的情况下，我们打开 设置 -> 更多设置 -> 开发者选项，开启开发者选项，找到下方调试选项，打开USB调试开关，此时下拉状态栏，如果看到有“已连接到USB调试”的会话，说明USB调试开启成功。

USB模式切换

有些Android设备没有USB调试开关,检查一下开发者选项中是否有USB模式切换选项，一般有3个选项，Device模式，Host模式和跳帽模式。连接到OTG USB接口,打开USB模式切换选项，选中Device模式，下拉状态栏，看是否有“已连接到USB调试”会话，如果有的话，说明开启成功。

WIFI调试

如果USB调试模式不可用的情况下也可以使用WIFI调试，WIFI调试分两种情况，一种是设备默认开启了5555端口，可以使用adb命令直连，另一种是需要开启5555端口后在使用adb连接。在同一局域网下，查看一下移动设备IP地址，在电脑端尝试使用adb命令连接下

如果连接成功，说明设备已开启了5555端口，可以直接通过电脑进行安装和调试程序。

如果提示连接超时，还需要下载Android-Terminal-Emulator客户端，使用命令开启5555端口

执行完命令后，在使用adb connect 命令连接即可。

USB有线连接传输速度比WIFI连接快很多，优先推荐使用USB连接，可以根据情况而定。
